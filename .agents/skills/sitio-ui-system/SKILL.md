# Sistema visual

O produto é uma ferramenta operacional familiar, simples e confiável — não um dashboard corporativo.

- Reutilize `AppShell`, `PageHeader`, `SectionCard`, `StatCard`, estados vazio/erro/loading, campos e itens de lista mobile.
- Centralize tokens de cor, tipografia, espaçamento, raio e sombra.
- Mobile usa cards/listas; desktop pode usar tabelas. Formulários não ocupam toda a largura no desktop.
- Não use gradientes chamativos, glassmorphism, sombras excessivas, gráficos 3D, cartões aninhados ou ícones decorativos.
- Um título e uma ação principal por página; labels claros em português e placeholders nunca substituem labels.

Leia `references/screen-patterns.md` e `references/design-tokens.md` quando for criar primitives ou telas.
