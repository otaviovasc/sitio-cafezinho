# UX de campo

Projete primeiro para 390x844 e valide em 360x800. A ação principal deve estar visível e os alvos importantes devem ter ao menos 44x44 CSS px. Use teclado decimal para litros e valores financeiros.

- Não dependa de hover, ícones ambíguos, tabelas horizontais ou selects enormes no celular.
- Salve apenas linhas preenchidas; preserve valores após erro e mostre o erro no campo.
- Inclua estados de loading, vazio, erro, sucesso e retry quando aplicável.
- Evite modais para formulários longos; no máximo uma ação principal por tela.

Leia `references/ux-checklist.md` antes de finalizar uma tela mobile.
